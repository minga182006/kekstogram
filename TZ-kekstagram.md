# ТЗ: «Кекстаграм»

## О проекте
Кекстаграм — сервис просмотра изображений. Пользователям предоставлена возможность загружать свои фотографии или просматривать фотографии, загруженные ранее другими пользователями.

## Описание функциональности

### 1. Загрузка нового изображения на сайт и заполнение информации о нем

#### 1.1. Загрузка нового изображения
- выбор файла с изображением для загрузки;
- изменение масштаба изображения;
- применение одного из заранее заготовленных эффектов;
- выбор глубины эффекта с помощью ползунка;
- добавление текстового комментария;
- добавление хэш-тегов.

#### 1.2. Выбор изображения для загрузки
- Загрузка осуществляется с помощью контрола загрузки файла `.img-upload__input`, стилизованного под букву «О» в логотипе.
- После выбора изображения (изменения значения поля `.img-upload__input`) показывается форма редактирования изображения.
- У элемента `.img-upload__overlay` удаляется класс `hidden`, у `body` задается класс `modal-open`.
- После выбора изображения нужно подставить его в форму редактирования вместо тестового изображения в блок предпросмотра и в превью эффектов.

#### 1.3. Закрытие формы редактирования
- Форма закрывается по кнопке `.img-upload__cancel` или по клавише `Esc`.
- Элементу `.img-upload__overlay` возвращается класс `hidden`.
- У элемента `body` удаляется класс `modal-open`.

### 2. Редактирование изображения и ограничения, накладываемые на поля

#### 2.1. Масштаб
- При нажатии на кнопки `.scale__control--smaller` и `.scale__control--bigger` должно изменяться значение поля `.scale__control--value`.
- Шаг изменения: 25.
- Начальное значение: 100%.
- Максимальное значение: 100%.
- Минимальное значение: 25%.
- При изменении значения `.scale__control--value` изображению внутри `.img-upload__preview` задается CSS `transform: scale(...)` (например, при 75% — `transform: scale(0.75)`).

#### 2.2. Наложение эффекта на изображение
- По умолчанию выбран эффект «Оригинал».
- На изображение можно наложить только один эффект.
- Интенсивность регулируется ползунком в слайдере (библиотека noUiSlider), значение пишется в `.effect-level__value`.
- При изменении уровня эффекта обновляется CSS-стиль картинки внутри `.img-upload__preview`.
- Значения по эффектам:
  - «Хром» — `grayscale(0..1)` с шагом 0.1;
  - «Сепия» — `sepia(0..1)` с шагом 0.1;
  - «Марвин» — `invert(0..100%)` с шагом 1%;
  - «Фобос» — `blur(0..3px)` с шагом 0.1px;
  - «Зной» — `brightness(1..3)` с шагом 0.1;
  - «Оригинал» — CSS-фильтр удаляется.
- При выборе «Оригинал» слайдер и его контейнер `.img-upload__effect-level` скрываются.
- При переключении эффектов насыщенность сбрасывается до начальной (100%), слайдер, CSS-стиль изображения и значение поля обновляются.

#### 2.3. Хэш-теги
- начинается с символа `#`;
- строка после решетки состоит из букв и чисел;
- не содержит пробелы, спецсимволы (`#`, `@`, `$`, `!` и т.п.) и символы пунктуации;
- хэш-тег не может состоять только из одной решетки;
- максимальная длина одного хэш-тега — 20 символов (включая решетку);
- хэш-теги нечувствительны к регистру (`#ХэшТег` и `#хэштег` — один и тот же тег);
- хэш-теги разделяются пробелами;
- один и тот же тег не может использоваться дважды;
- нельзя указать больше пяти хэш-тегов;
- хэш-теги необязательны;
- если фокус в поле ввода хэш-тега, `Esc` не должен закрывать форму редактирования.

#### 2.4. Комментарий
- комментарий необязателен;
- длина комментария — не более 140 символов;
- если фокус в поле комментария, `Esc` не должен закрывать форму редактирования.

### 3. Отправка данных на сервер

#### 3.1. Отправка формы
- После заполнения данных, при нажатии «Отправить», все данные формы (включая изображение) отправляются AJAX-запросом на `https://29.javascript.htmlacademy.pro/kekstagram` методом `POST` с типом `multipart/form-data`.
- На время отправки кнопка «Опубликовать» блокируется.

#### 3.2. Валидация перед отправкой
- Страница проверяет корректность введенных значений.
- Если данные не соответствуют ограничениям из п. 2.3 и 2.4, отправка не происходит.
- Для проверки используется Pristine.
- Должны быть разные сообщения ошибок минимум для случаев:
  - введен невалидный хэш-тег;
  - превышено количество хэш-тегов;
  - хэш-теги повторяются.
- Количество одновременно показываемых сообщений для поля определяется разработчиком.

#### 3.3. Состояние после успешной отправки
- Форма редактирования закрывается;
- введенные данные и контроль фильтра приходят в исходное состояние:
  - масштаб 100%;
  - эффект «Оригинал»;
  - поля хэш-тегов и комментария очищаются;
  - поле загрузки фотографии очищается.

#### 3.4. Сообщение об успехе
- При успешной отправке показывается блок `.success` (из шаблона `template`), размещенный перед закрывающим тегом `</body>`.
- Сообщение исчезает по нажатию на `.success__button`, по клавише `Esc` или по клику вне блока сообщения.

#### 3.5. Сообщение об ошибке
- При ошибке отправки показывается блок `.error` (из шаблона `template`), размещенный перед `</body>`.
- Сообщение исчезает по нажатию на `.error__button`, по клавише `Esc` или по клику вне блока сообщения.
- Введенная пользователем информация сохраняется для повторной отправки.

#### 3.6. Закрытие по отмене
- Нажатие на `.img-upload__cancel` приводит к закрытию формы и возврату полей в исходное состояние (как в п. 3.3).

### 4. Просмотр загруженных изображений

#### 4.1. Загрузка изображений
- Изображения других пользователей загружаются сразу после открытия страницы с `https://29.javascript.htmlacademy.pro/kekstagram/data`.

#### 4.2. Ошибка загрузки данных
- Если загрузка данных не удалась, показывается соответствующее сообщение.
- Дизайн блока сообщения нужно продумать самостоятельно.

#### 4.3. Отрисовка миниатюр
- Все загруженные изображения показываются в виде миниатюр.
- DOM-элемент миниатюры формируется на основе шаблона `#picture` в элементе `template`.

#### 4.4. Полноразмерный просмотр
- По клику на миниатюру показывается блок `.big-picture` с полноразмерным изображением, лайками и комментариями.
- Элементу `body` задается класс `modal-open`.
- Описание изображения подставляется в соответствующие элементы разметки.

#### 4.5. Закрытие полноэкранного режима
- Режим закрывается по иконке крестика `.big-picture__cancel` или по клавише `Esc`.
- У `body` удаляется класс `modal-open`.

#### 4.6. Вывод комментариев
- Сразу после открытия в полноэкранном режиме отображается не более 5 комментариев.
- Количество показанных и общее число выводятся в `.social__comment-count`.
- Комментарии выводятся в `.social__comments`.
- Каждый комментарий оформляется отдельным элементом списка:
  - аватар в `.social__picture` (имя автора в `alt`);
  - текст в `.social__text`.

#### 4.7. Дозагрузка комментариев
- Дополнительные комментарии показываются по кнопке `.comments-loader`.
- За одно нажатие выводится не более 5 новых комментариев.
- Счетчик в `.social__comment-count` обновляется.

#### 4.8. Скрытие кнопки дозагрузки
- Если все комментарии показаны, `.comments-loader` скрывается (добавляется `hidden`).

### 5. Фильтрация изображений от других пользователей

#### 5.1. Доступные фильтры
- «По умолчанию» — фотографии в исходном порядке с сервера;
- «Случайные» — 10 случайных неповторяющихся фотографий;
- «Обсуждаемые» — фотографии, отсортированные по убыванию количества комментариев.

#### 5.2. Поведение блока фильтров
- Блок фильтров по умолчанию скрыт.
- Показывается только после получения данных об изображениях от других пользователей.

#### 5.3. Ограничение частоты отрисовки
- При переключении фильтров отрисовка должна происходить не чаще 1 раза в 500 мс (устранение дребезга).
